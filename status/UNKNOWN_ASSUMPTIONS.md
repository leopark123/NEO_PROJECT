# ⚠️ 未知假设登记表

> **核心原则**：不确定性是第一等公民  
> **规则**：任何代码中的默认值/阈值/策略，如果无法在 spec/evidence 中找到来源，必须登记在此

---

## 登记规则

Claude Code 在实现过程中，如果遇到以下情况，**必须在此登记**，不得自行决定：

1. spec 中未明确定义的数值（阈值、超时、缓冲区大小等）
2. 多种实现方式中选择了其中一种
3. 边界情况的处理策略
4. 任何"合理推断"的决定

---

## 登记格式

```markdown
### UA-001 [状态]
- **位置**：src/xxx/xxx.cs 第 N 行
- **内容**：`const int BufferSize = 4096`
- **原因**：spec 未定义缓冲区大小，采用常见默认值
- **风险等级**：🟡 中（可能影响性能）
- **需要确认**：是否有设备端约束？
- **状态**：🟡 待确认 / ✅ 已确认 / ❌ 需修改
```

## 风险等级说明

| 等级 | 含义 | 处理方式 |
|------|------|----------|
| 🟢 低 | 纯技术实现细节，不影响功能正确性 | 可继续开发，后续优化 |
| 🟡 中 | 可能影响性能或边界行为 | 需要指挥官确认后继续 |
| 🔴 高 | 可能影响医学正确性或数据完整性 | **必须停止，等待确认** |
